# Contributing

## 目标

本仓库接收的是“通用 AI 自动化抓包+测试工具”的方法论、抽象设计和提示词工程改进，而不是任何单一业务系统的配置转储。

## 贡献顺序

新增能力时按以下顺序提交：

1. 先补能力文档
2. 再补相关 prompt
3. 再补扩展规范或评估标准

如果只改 prompt，也需要同步更新它的输入契约或评估规则。

## 必须遵守的规则

- 不提交真实流量样本
- 不提交真实密钥、令牌、Cookie、Header 值
- 不提交供应商专属路径、状态码约定、业务字段字典
- 不把某个具体公司的业务模型包装成“通用模型”
- 不写必须依赖某种框架才能成立的规范

## 文档风格

- 中文为主，术语保留英文原词
- 优先写能力边界和决策规则
- 示例应使用中性命名
- 同一个概念全仓保持一致命名

## Prompt 变更要求

每个 prompt 文件都必须包含：

1. 角色定位
2. 输入契约
3. 输出契约
4. 决策规则
5. 禁止事项
6. 失败处理
7. 示例输入输出
8. 评估标准

## 评审标准

- 是否保持供应商无关
- 是否降低实现歧义
- 是否便于 AI 稳定输出结构化结果
- 是否避免把私有知识误写为通用原则

## 提交建议

- 一个提交只覆盖一个能力面
- 标题使用“能力 + 动作”形式
- 在说明里写出修改影响的文档和术语
