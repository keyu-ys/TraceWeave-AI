# Contributing

## 贡献目标

TraceWeave AI 接受的是“可迁移、可执行、可验证”的方法论贡献，而不是某个具体业务系统的经验堆积。

合格贡献至少满足三点：

1. 能映射到一个通用工具能力
2. 能说明输入、输出、决策规则和失败模式
3. 不携带任何真实敏感流量、私有协议或供应商耦合细节

## 贡献顺序

新增或改进内容时，按这个顺序提交：

1. 先更新能力或算法文档
2. 再更新相关 Prompt 契约
3. 最后更新实施手册或路线图

如果一个新想法无法先写成能力文档，通常意味着问题定义还不够清楚。

## 贡献标准

- 优先补“算法缺口”，不要优先补“目录覆盖面”
- 对任何策略都要写清楚 deterministic 部分和 AI 参与部分
- 伪代码、公式、阈值和决策表优先于泛泛描述
- 对低置信推断必须给出人工确认机制
- 所有相对链接必须可直接用于 GitHub 浏览

## 不接受的内容

- 真实抓包样本或半脱敏样本
- 真实 token、cookie、secret、设备标识
- 供应商特定接口路径、状态码语义、环境配置
- 依赖某一私有业务字段才能成立的算法
- 只写目标、不写输入输出和决策规则的浅层文档

## 文档写法

- 中文主写，术语保留英文原词
- 优先结构化 Markdown
- 表格、决策表、流程图、伪代码优先
- 对不确定结论使用 `待人工确认`
- 反例和失败模式要与正向方法并列出现

## Pull Request 自检

提交前至少检查：

1. 是否引入了任何具体业务痕迹
2. 是否只讲“做什么”而没有讲“怎么做”
3. 是否给出了输入、输出、步骤、失败模式
4. 是否有过度参数化、AI 越权或敏感信息外泄风险
5. 是否更新了受影响的交叉链接
