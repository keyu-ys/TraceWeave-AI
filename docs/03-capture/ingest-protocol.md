# 抓包入库协议

## 目标

为任意采集源提供统一的入库契约，使后续分析和回放不依赖采集方式。

## 最小输入单元

一次请求交换应至少包含：

- 请求方法
- 完整 URL
- 请求 Header
- 请求 Body 预览或全文
- 响应状态
- 响应 Header
- 响应 Body 预览或全文
- 开始时间、结束时间、耗时
- 采集源标识

## 设计要求

- 原始值与标准化值分层保存
- 允许 Body 截断，但要标明截断状态
- 允许无响应场景，例如超时或断连
- 允许无 JSON 场景，例如二进制或纯文本
- 禁止在采集层嵌入供应商特定业务语义

## 入库结果

采集层输出的是 `CapturedRequest`、`CapturedResponse` 和它们组成的 `NormalizedExchange`。
